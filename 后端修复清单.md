# 后端需要修复的问题

## 1. 读者大厅暴露未发布小说（最严重）

以下三个接口查询小说时，需要加上 `isPublished = true` 过滤条件：
- `GET /reader/browse` — 读者大厅推荐列表
- `GET /reader/genre/:genre` — 按题材筛选
- `GET /reader/search?q=xxx` — 搜索

示例 SQL：
```sql
-- 错误的（当前）：
SELECT * FROM novels ORDER BY updated_at DESC;

-- 正确的：
SELECT * FROM novels WHERE isPublished = true ORDER BY updated_at DESC;
```

## 2. AI 生成大纲返回 0 字

如果地址配对了还是 0 字，检查后端：
- `/api/ai/generate-outline` 接口是否正确返回 `{ code: 200, data: { content: "..." } }` 格式
- AI 模型提供商的 API Key 是否有效、余额是否充足
- 流式接口 `/api/ai/generate-chapter-stream` 是否实现了 SSE 格式（event: / data: 格式）

## 3. 安全问题（建议）

- localStorage 存 token 有 XSS 风险，建议后端设置 httpOnly cookie
- 对 AI 生成接口加 rate limiting，防止滥用刷 API 费用
